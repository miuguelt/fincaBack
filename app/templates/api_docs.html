🔐 Autenticación

Sistema de Autenticación JWT

La API utiliza tokens JWT para la autenticación. Todos los endpoints (excepto login) requieren un token válido.

Métodos de Autenticación:
  • Header Authorization: Authorization: Bearer <token>
  • Cookie: access_token_cookie (automático)
POST /api/v1/auth/login

Autenticar usuario y generar tokens JWT

Request Body:
{ "identification": 12345678, "password": "password123" }
Response (200):
{ "message": "Autenticación exitosa", "user": { "id": 1, "fullname": "Juan Pérez", "role": "Instructor" }, "access_token": "eyJ0eXAiOiJKV1QiLCJhbGciOiJIUzI1NiJ9...", "refresh_token": "eyJ0eXAiOiJKV1QiLCJhbGciOiJIUzI1NiJ9..." }
Códigos de Estado:
200 Autenticación exitosa
401 Credenciales incorrectas
404 Usuario no encontrado
400 Datos inválidos

👥 Gestión de Usuarios

Endpoints de Usuarios
GET /api/v1/users/
🔒 Requiere autenticación

Obtener lista de usuarios con filtros opcionales

Parámetros de consulta:
  • fullname - Filtrar por nombre (búsqueda parcial)
  • role - Filtrar por rol (Aprendiz, Instructor, Administrador)
  • status - Filtrar por estado (true/false)
Ejemplo de uso:
GET /api/v1/users/?role=Instructor&status=true
Response (200):
{ "users": [ { "id": 1, "identification": 12345678, "fullname": "Juan Pérez", "email": "juan@example.com", "role": "Instructor", "status": true } ], "total": 1 }
POST /api/v1/users/
🔒 Requiere rol Administrador

Crear un nuevo usuario en el sistema

Request Body:
{ "identification": 87654321, "fullname": "María García", "password": "password123", "email": "maria@example.com", "phone": "3001234567", "address": "Calle 123 #45-67", "role": "Instructor", "status": true }
Códigos de Estado:
201 Usuario creado
403 Sin permisos
409 Datos duplicados
400 Datos inválidos

🐄 Gestión de Animales

Inventario Ganadero
GET /api/v1/animals/
🔒 Requiere autenticación

Obtener inventario completo de animales con filtros avanzados

Parámetros de consulta:
  • record - Filtrar por registro del animal
  • breeds_id - Filtrar por ID de raza
  • sex - Filtrar por sexo (Hembra, Macho)
  • status - Filtrar por estado (Vivo, Vendido, Muerto)
  • min_weight - Peso mínimo en kg
  • max_weight - Peso máximo en kg
Ejemplo de uso:
GET /api/v1/animals/?sex=Hembra&status=Vivo&min_weight=400
Response (200):
{ "animals": [ { "idAnimal": 1, "birth_date": "2023-01-15", "weight": 450, "breed": { "id": 1, "breed": "Holstein", "species_id": 1 }, "sex": "Hembra", "status": "Vivo", "record": "BOV-001-2023", "idFather": 2, "idMother": 3 } ], "total": 1 }
POST /api/v1/animals/
🔒 Requiere rol Administrador

Registrar un nuevo animal en el inventario

Request Body:
{ "sex": "Hembra", "birth_date": "2023-06-15", "weight": 380, "record": "BOV-002-2023", "status": "Vivo", "breeds_id": 1, "idFather": 2, "idMother": 3 }

💡 Ejemplos de Uso

Casos de Uso Comunes
1. Flujo completo de autenticación:
# 1. Login curl -X POST {{ config.get('API_BASE_URL', 'https://localhost:8081/api/v1') }}/auth/login \ -H "Content-Type: application/json" \ -d '{ "identification": 12345678, "password": "password123" }' # 2. Usar el token en requests posteriores curl -X GET {{ config.get('API_BASE_URL', 'https://localhost:8081/api/v1') }}/users/ \ -H "Authorization: Bearer YOUR_TOKEN_HERE"
2. Crear un nuevo animal:
curl -X POST {{ config.get('API_BASE_URL', 'https://localhost:8081/api/v1') }}/animals/ \ -H "Content-Type: application/json" \ -H "Authorization: Bearer YOUR_TOKEN_HERE" \ -d '{ "sex": "Hembra", "birth_date": "2023-06-15", "weight": 380, "record": "BOV-002-2023", "breeds_id": 1 }'
3. Filtrar animales por criterios:
# Obtener todas las hembras vivas con peso mayor a 400kg curl -X GET "{{ config.get('API_BASE_URL', 'https://localhost:8081/api/v1') }}/animals/?sex=Hembra&status=Vivo&min_weight=400" \ -H "Authorization: Bearer YOUR_TOKEN_HERE"

⚠️ Códigos de Error

Referencia de Códigos de Estado HTTP
Códigos de Éxito:

200 OK - Solicitud exitosa

201 Created - Recurso creado exitosamente

Códigos de Error del Cliente:

400 Bad Request - Datos inválidos

401 Unauthorized - Token requerido o inválido

403 Forbidden - Permisos insuficientes

Más Códigos de Error:

404 Not Found - Recurso no encontrado

409 Conflict - Error de integridad

500 Internal Server Error - Error del servidor

Ejemplo de respuesta de error:
{ "message": "Token JWT requerido", "error": "Missing Authorization header" }
